The Problem

Enterprises won't deploy autonomous agents to untrusted infra because:

  • Plaintext exposure: prompts, memory, model weights are visible to the host. Non-starter for defence, finance, healthcare.
  • No enforced expiry: you can't mathematically make an agent stop existing after a deadline.
  • No deletion proof: you can't prove the agent actually destroyed its secrets. You just hope the host isn't lying.

TEEs (SGX, TDX) try to fix this but are repeatedly broken and cannot prove post-mortem erasure. We need a cryptographic guarantee, not a hardware promise.

What CHRONOS Does

Three guarantees, composited into one agent lifecycle.

  • Plaintext blindness: Agent runs on TFHE-encrypted data. The private key is encrypted under a VDF output, locked until deadline.
  • Unforgeable time-lock: A Wesolowski VDF over an MPC-generated RSA modulus forces sequential work. No shortcut – not even the agent can speed it up.
  • Attestable software erasure: At deadline the agent unlocks the key, uses it once, overwrites the memory region, and produces a Groth16 SNARK proving: VDF valid, key correct, memory zeroised. Proof is 192 bytes, verifiable in 15ms.

We call it “attestable software erasure” under an explicit Exclusivity Assumption. We don't claim physics – we're honest engineers.

Architecture – 5 Phase Lifecycle

[Setup] → [Operation] → [Dead Man's Switch] → [Erasure] → [Attestation]

  • Setup: Multi‑party modulus generation (Diogenes) → FHE keygen → encrypt private key under VDF → delete plaintext key.
  • Operation: FHE inference on encrypted inputs. Background thread computes the sequential VDF.
  • Dead Man's Switch: drand beacon (BLS threshold) triggers deadline. Agent cannot skip it.
  • Erasure: Decrypt key once, triple‑pass memory overwrite, generate ZK proof of zeroisation.
  • Attestation: The proof is publicly verifiable. No trust in the agent.

The Moat – Why This Is Defensible

We didn't just glue libraries; we did the math.

  • ZK compositeness proof (Schnorr-style over Poseidon) – verifier knows the VDF modulus is composite without learning factors.
  • UC composition theorem – first formal proof that an ephemeral FHE agent securely realises an ideal functionality in the hybrid model.
  • Formal security reductions – time‑bound key release under factoring & VDF sequentiality; erasure attestation under Groth16 knowledge soundness.
  • Real hardware benchmarks – CPU, GPU (A100), FPGA (Alveo U280), vs SGX baseline. All in the paper.

This is cryptographic IP, not a prompt wrapper. That’s our moat.

Market – Not AI Apps, Infrastructure

We're the cryptographic substrate for agents in regulated environments.

  • Defence/intelligence: mission‑scoped analysis agents that must self‑destruct after 72h.
  • DePIN networks: nodes proving they executed a task and shredded the data.
  • Financial services: encrypted audit agents with time‑bound validity.
  • Enterprise SaaS: “zero‑knowledge agent processing” as a compliance feature.

Confidential computing market: $54B+ by 2026. We target the slice that refuses to trust silicon.

Competition

  • TEEs (SGX, SEV, TDX): fast but broken repeatedly, can't prove erasure, require hardware trust.
  • Stateless AI: avoids retention but can't enforce time limits.
  • Crypto‑shredding: deletes remote key; no proof of local deletion.

CHRONOS is the only approach that gives cryptographic time‑binding + attestable erasure without a hardware root of trust. New category.
