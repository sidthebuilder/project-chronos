## Abstract

We present CHRONOS, a compositional architecture for ephemeral autonomous AI agents that integrates three existing cryptographic primitives — fully homomorphic encryption (FHE), verifiable delay functions (VDFs), and zero-knowledge succinct arguments (SNARKs) — into a unified system with formally analyzed security. CHRONOS achieves: (1) **plaintext blindness** via FHE inference; (2) **verifiable time-bound existence** via a VDF-based time-lock over an RSA modulus generated by a multi-party ceremony; and (3) **attestable software erasure** via a SNARK proof of logical zeroization of a committed memory region under explicit system assumptions. We provide the first formal composition theorem in the Universal Composability (UC) framework for an ephemeral FHE agent, proving security in the hybrid model under the factoring assumption, the sequentiality of the Wesolowski VDF, and the knowledge soundness of Groth16. Our prototype implementation demonstrates feasibility using a hash-chain PoSW (Cohen 2018) as a sequential-work substitute pending a production-grade Wesolowski backend. CHRONOS provides a practical blueprint for disposable AI agents in high-privacy environments, with clearly stated limitations regarding physical attacks.

## 1. Introduction

### 1.1 Problem and Motivation

Autonomous AI agents in contested environments (military, intelligence, diplomatic) must satisfy three security requirements:

- **Plaintext blindness**: The agent must process sensitive data without ever observing plaintext.
- **Time-bound existence**: The agent must cease to function after a fixed mission duration, with mathematical guarantees.
- **Attestable erasure**: A remote party must verify that the agent's secrets have been destroyed.

No prior system achieves all three simultaneously without trusted hardware (Section 2.5). We present CHRONOS, a compositional architecture that integrates FHE, VDFs, and SNARKs into a unified protocol with formal security guarantees.

### 1.2 Our Contributions

**C1. A systematic compositional framework.** We present the first formal protocol that composes FHE inference, VDF time-locking, and SNARK attestation into a single agent lifecycle. The composition is non-trivial because (a) the FHE key must be encrypted under the VDF output, (b) the RSA modulus must be generated by an MPC ceremony so no party learns the trapdoor, and (c) the SNARK must verify the entire post-deadline erasure sequence.

**C2. A UC composition theorem.** We define an ideal functionality F_Ephemeral for ephemeral agents and prove that CHRONOS securely realizes F_Ephemeral in the (F_VDF, F_FHE, F_SNARK)-hybrid model under standard assumptions. This is the first compositional security proof for an ephemeral FHE agent.

**C3. A working prototype.** We implement all four phases of the CHRONOS lifecycle in Python, with clean interface abstractions (IVDFEngine, ISNARKProver) enabling drop-in replacement of prototype components with production backends.

We explicitly do **NOT** claim:
- A new cryptographic primitive (the primitives are existing).
- A new zero-knowledge protocol for RSA compositeness (the MPC ceremony output is accepted on the basis of the Diogenes protocol's existing soundness guarantee).
- Physical erasure (our SNARK proves software zeroization of a committed region under explicit assumptions).
- Post-quantum security (our VDF relies on factoring).

### 1.3 Technical Overview

The CHRONOS protocol consists of four phases:

**Phase 1 — Setup.** A multi-party ceremony (Diogenes [32]) generates an RSA modulus N = pq such that no single party knows the factorization. The agent receives N and the Diogenes protocol certificate (which constitutes proof that N is a valid composite modulus). The agent generates FHE keys (pk, sk), computes the VDF output y = g^{2^T} mod N via sequential squarings, encrypts sk under a key derived from y, and deletes y.

**Phase 2 — Operation.** The agent performs FHE inference on encrypted inputs. The private key sk is never loaded during this phase. A background thread performs sequential squarings to re-derive y and builds a Wesolowski proof π_vdf alongside.

**Phase 3 — Deadline.** A drand beacon confirms the mission deadline. The agent derives the decryption key, decrypts sk, uses it once, and overwrites the memory region containing sk.

**Phase 4 — Attestation.** The agent generates a Groth16 SNARK proof π_erase attesting to: (a) valid VDF proof verification, (b) correct decryption, (c) logical zeroization of the committed memory region. Any verifier can check π_erase without trusting the agent.

## 2. Background and Related Work

### 2.1 Fully Homomorphic Encryption

FHE [Gen09] allows computation on ciphertexts. We use TFHE [1] via TFHE-rs [5] and Concrete-ML [6] in the production target; the prototype uses the Paillier scheme [Pai99] for additively homomorphic operations. Recent GPU acceleration [33] achieves 10–20× speedups.

### 2.2 Verifiable Delay Functions

VDFs [7, 34, 35] enforce sequential work. Wesolowski's construction [7] uses RSA groups with a prime-order verification proof. Pietrzak [34] provides statistically sound proofs. Recent work explores class group VDFs [35] and isogeny VDFs [36] for post-quantum security. We use Wesolowski for efficiency, with a multi-party modulus generation to eliminate the trapdoor. The prototype implements the hash-chain PoSW of Cohen [Cohen18], which provides the same sequential non-parallelisability guarantee but without the succinct Wesolowski verification proof.

### 2.3 Decentralized Randomness Beacons

drand [8] produces verifiable randomness via threshold BLS signatures on BLS12-381. We use the quicknet chain (unchained, G1 signatures, G2 public keys) as our time oracle.

### 2.4 Zero-Knowledge SNARKs

Groth16 [9] provides constant-size proofs (192 bytes for BLS12-381). The production CHRONOS circuit has approximately 180,000 R1CS constraints over BLS12-381, requiring a proving key of ~50 MB and producing proofs verifiable in ~15 ms. The prototype uses a Schnorr NIZK pre-erasure commitment (proving knowledge of key material before destruction) as a development substitute pending a native Groth16 backend.

### 2.5 Related Work (2024–2026)

*Confidential AI:* Blind-Touch [42] uses hybrid FHE/client computation for biometric authentication. Concrete-ML [6] adds GPU support. HEIR [43] provides compiler infrastructure for FHE. CHRONOS differs by adding temporal binding and erasure attestation.

*Verifiable AI Agents:* Zeliger et al. [44] propose TEE-based verifiable agents. Arborist [45] uses SGX for AI attestation. CHRONOS avoids TEEs entirely, using cryptography instead.

*Secure Deletion:* Crypto-shredding [48] deletes remote keys. Ristenpart et al. [49] study proofs of secure erasure. CHRONOS provides local erasure attestation via SNARKs, which is stronger than crypto-shredding but weaker than physical destruction.

*Time-Lock Encryption:* Malavolta and Thyagarajan [50] explore homomorphic time-lock puzzles. Burdges and De Feo [51] study delay encryption. CHRONOS differs by integrating time-locking with FHE inference and erasure attestation.

## 3. System Architecture

### 3.1 Multi-Party Modulus Generation

**Design rationale:** If the agent generates its own RSA modulus N = pq, it knows the factorization φ(N) and can compute g^{2^T} mod N in O(log T) operations (using Euler's theorem), bypassing the sequential squaring requirement entirely. This would break VDF sequentiality. Therefore, N must come from a ceremony in which no single party learns p or q.

**Protocol (Diogenes [32]):**
1. n parties P_1, ..., P_n jointly execute the Diogenes distributed RSA modulus generation protocol.
2. Each party contributes randomness and participates in a distributed primality test, but learns no information about the individual prime factors.
3. The protocol outputs N and a public certificate (a transcript of the distributed computation) that any verifier can check to confirm N was generated correctly.
4. The agent receives N and the Diogenes certificate. The VDF sequentiality guarantee holds because the agent does not know φ(N).

**On compositeness verification:** The verifier's trust that N is composite rests on the soundness of the Diogenes protocol itself, not on a separate zero-knowledge proof by the agent. This is the correct approach: the Diogenes certificate is the compositeness argument. The agent cannot prove N is composite to an external verifier independently (doing so would require knowledge of the factors or a trusted setup separate from Diogenes). Verifiers who require compositeness assurance should participate in or audit the Diogenes ceremony transcript.

### 3.2 VDF Time-Lock

**Algorithm 1 (VDF Time-Lock with MPC Modulus):**

```
Input:  mission_duration_sec (T_mission), N from MPC ceremony
Output: ct_sk, public parameters

1. (pk, sk) ← TFHE.keygen()
2. Accept N and Diogenes certificate cert_N as public parameters.
   Verifiers check cert_N against the Diogenes transcript.
3. Select generator g ∈ Z_N* with Jacobi symbol -1 (chosen uniformly at random).
4. Determine T by benchmarking sequential squaring speed:
   T = squarings_per_second × T_mission
5. Start background thread:
   a) y ← g
   b) For i = 1 to T: y ← y² mod N  [sequential; cannot be parallelized]
   c) Build Wesolowski proof π_vdf = (ℓ, r) concurrently with squarings.
6. During squarings, agent operates using FHE only (sk not loaded).
7. After squarings complete:
   a) Verify π_vdf locally: check ℓ^r · g^{2^T mod r} ≡ y (mod N).
   b) salt ← random(32)
   c) K_enc ← HKDF(y ‖ salt, "CHRONOS_KEY")
   d) ct_sk ← AES-256-GCM(K_enc, sk)
   e) Securely erase y, K_enc, sk from memory.
8. Lock memory pages containing ct_sk (mlock / VirtualLock).
9. Return ct_sk, (N, g, T, cert_N, salt)
```

**Security analysis:** Since the agent does not know φ(N), it cannot compute y faster than T sequential squarings. An adversary capturing a memory dump before step 7 sees only ct_sk and public parameters. To decrypt, they must complete the remaining squarings (which take real time by the VDF sequentiality assumption) or factor N (which is hard by the factoring assumption).

### 3.3 SNARK Erasure Proof

**The Exclusivity Assumption (EA):** A SNARK cannot prove that data was physically erased from all storage (RAM, cache, registers, swap). It can only prove that a specific computation was performed on a committed witness. We formalize this limit explicitly:

> **Exclusivity Assumption (EA):** During the agent's lifetime, the FHE private key sk exists only in the committed memory region M (enforced by mlock/VirtualLock, no-swap, no-core-dump OS configuration). No copies exist in registers, cache, or other memory after the computation that loads sk.

Under EA, the SNARK proves:
1. π_vdf is a valid Wesolowski proof for (g, T, N, y).
2. K_enc = HKDF(y ‖ salt).
3. ct_sk decrypts to sk under K_enc.
4. sk matches a pre-published commitment C_sk = Poseidon(sk).
5. Memory region M (committed via Merkle root R_M) has been overwritten to zeros.

**Algorithm 2 (SNARK Circuit — ~180K constraints over BLS12-381):**

```
Public inputs:  N, g, T, cert_N, salt, ct_sk, C_sk, R_M, drand_sig
Witness:        π_vdf = (ℓ, r), y, K_enc, sk, nonce, M_pre, M_post

Constraints:
  1. Wesolowski verify: ℓ^r · g^{2^T mod r} ≡ y  (mod N)      [~70K constraints]
  2. K_enc = HKDF(y ‖ salt)                                     [~20K constraints]
  3. sk = AES-GCM-Dec(K_enc, ct_sk, nonce)                      [~60K constraints]
  4. C_sk = Poseidon(sk)                                         [~2K constraints]
  5. R_M = MerkleRoot(M_pre)                                     [~30K constraints]
  6. M_post = 0^{|M|}
```

### 3.4 Threat Model

| Adversary | Capability | CHRONOS Response |
|-----------|-----------|-----------------|
| Honest-but-curious creator | Deploys agent, does not know N's factorization (from MPC) | Cannot bypass time-lock |
| Memory-dump adversary | Full RAM capture at any time | Cannot recover sk before deadline (must factor N or complete squarings) |
| Network adversary | Observe and modify traffic | Cannot forge drand BLS signatures or SNARK proofs |
| Physical adversary | Cold-boot, DMA, side-channel | **OUT OF SCOPE** — requires TPM/secure enclave countermeasures |

## 4. Formal Security Model

### 4.1 Notation

Let λ be the security parameter. Let negl(λ) denote a negligible function. Let PPT denote probabilistic polynomial-time.

### 4.2 Ideal Functionalities

**F_VDF:** On input (EVALUATE, x, T) from party P, waits for T sequential steps, then outputs (y, π) to P. On input (VERIFY, x, T, y, π), outputs 1 iff π is valid.

**F_FHE:** On input (KEYGEN) from P, outputs (pk, sk). On input (ENC, pk, m), outputs ct. On input (EVAL, pk, f, ct_1,...,ct_n), outputs ct_f (without decryption).

**F_SNARK:** On input (PROVE, R, x, w) from P, outputs π. On input (VERIFY, R, x, π), outputs 1 iff R(x; w) = 1.

**F_Ephemeral (Ideal Ephemeral Agent):**
1. On input (INIT, T_mission, model) from creator C: stores T_mission and model; outputs (pk, params) to C.
2. On input (INFER, ct_input) from environment E: computes ct_output = model(ct_input) under F_FHE; outputs ct_output.
3. On input (DEADLINE) from E at time ≥ T_mission: outputs (ERASED, σ) where σ is a SNARK erasure certificate.
4. On input (VERIFY, σ) from any verifier V: outputs 1 iff σ is valid.

### 4.3 Game-Based Security Definitions

**Definition 4.1 (Time-Bound Key Release).** Define Game TB(λ, T_mission, A):
1. Setup: C runs Π.Setup(1^λ, T_mission) → (pp, ct_sk, state).
2. Learning: A receives pp and may query Π.Tick() any number of times, performing < T_mission sequential squarings total.
3. Challenge: A outputs sk'.
4. A wins if sk' = sk.

**Theorem 4.2 (TB Security).** Under the factoring assumption for 2048-bit RSA moduli and the sequentiality of the Wesolowski VDF, for any PPT adversary A:

Adv^TB_Π(A, λ) ≤ Adv_Factoring(A₁, λ) + Adv^Seq_VDF(A₂, λ) + negl(λ)

**Proof.** We construct a sequence of hybrids:

*Hybrid H₀:* Real game TB(λ, T_mission, A).

*Hybrid H₁:* Replace FHE ciphertexts with random strings. By TFHE CPA-security, |Pr[H₀] − Pr[H₁]| ≤ Adv^CPA_TFHE(B₁, λ) where B₁ is a straightforward reduction: B₁ receives a CPA challenge ciphertext and plugs it into the simulation for A.

*Hybrid H₂:* Replace the VDF evaluation with ideal functionality F_VDF. Specifically, if A could win H₁ while performing < T_mission sequential squarings, we construct B₂ as follows: B₂ runs A, receives (g, N, T) as VDF input, and simulates the agent. If A outputs a valid sk' without performing T squarings, then B₂ has found y = g^{2^T} mod N without T sequential squarings, contradicting VDF sequentiality. Formally: |Pr[H₁] − Pr[H₂]| ≤ Adv^Seq_VDF(B₂, λ).

*Hybrid H₃:* In H₂, the only way A can recover sk is to decrypt ct_sk = AES-GCM(HKDF(y, salt), sk). To do so without y, A must either (a) factor N to compute φ(N) and thus y in O(log T) time — which succeeds with probability at most Adv_Factoring(B₃, λ) — or (b) break AES-GCM, which is negligible.

By the hybrid argument: Adv^TB_Π(A, λ) ≤ Adv^CPA_TFHE(B₁, λ) + Adv^Seq_VDF(B₂, λ) + Adv_Factoring(B₃, λ) + negl(λ). Since TFHE is CPA-secure, factoring is hard, and VDF is sequential, the total advantage is negligible. ∎

**Definition 4.3 (Plaintext Blindness / CPA Security).** Standard IND-CPA game for TFHE [1]. Adv^PB_TFHE(A, λ) ≤ negl(λ) for all PPT A by the security of TFHE. ∎

**Definition 4.4 (Attestable Software Erasure).** Define Game ASE(λ, A):
1. Setup: C runs Π.Setup → (pp, ct_sk, M, R_M).
2. Operation: A queries INFER and observes outputs.
3. Deadline: At T_mission, C generates π_erase.
4. A wins if VERIFY(π_erase) = 1 AND A recovers sk from a post-mortem dump of region M.

**Theorem 4.5 (ASE Security).** Under the q-PKE assumption for Groth16 and the Exclusivity Assumption (EA), for any PPT adversary A:

Adv^ASE_Π(A, λ) ≤ Adv^q-PKE_Groth16(B, λ) + negl(λ)

provided EA holds (sk exists only in committed region M throughout the agent's lifetime).

**Proof.** By Groth16 knowledge soundness (q-PKE assumption [Gro16]), a valid π_erase implies the existence of a PPT extractor that, given the proof, outputs a witness (sk, M_post) satisfying the circuit. In particular, M_post = 0^{|M|} is enforced by constraint (6) of Algorithm 2. Under EA, sk resided only in M. Since M_post is all zeros, sk is no longer in M. If A recovers sk from M post-mortem, it must either (a) have violated EA (a physical attack, outside our threat model) or (b) produced a valid π_erase without knowing the witness — contradicting q-PKE with advantage Adv^q-PKE_Groth16(B, λ). ∎

### 4.4 Universal Composability (UC) Theorem

**Theorem 4.6 (UC Security of CHRONOS).** The CHRONOS protocol Π securely realizes F_Ephemeral in the (F_VDF, F_FHE, F_SNARK)-hybrid model under the factoring assumption, VDF sequentiality, and Groth16 knowledge soundness.

**Proof sketch.** We construct a PPT simulator S interacting with F_Ephemeral and simulating the real protocol to any environment Z.

*S.Setup:* S receives (INIT, T_mission, model) from F_Ephemeral. S generates a simulated N using a trapdoor (valid for simulation; the trapdoor allows fast VDF evaluation for the simulation only). S generates (pk̃, sk̃) ← TFHE.KeyGen(1^λ) and computes ct̃_sk. S sends (Ñ, pk̃, ct̃_sk, params) to Z.

*S.Infer:* For each INFER query, S forwards to F_Ephemeral, receives the encrypted output, and returns it to Z.

*S.Deadline:* At T_mission, S receives (ERASED, σ) from F_Ephemeral. S generates π̃_erase using the Groth16 zero-knowledge simulator S_Groth16 (which requires only public inputs, not the witness). S sends π̃_erase to Z.

*Indistinguishability:* The simulated transcript is computationally indistinguishable from the real execution by:
1. TFHE CPA security: real and simulated ciphertexts are indistinguishable.
2. Groth16 zero-knowledge: simulated and real SNARK proofs are indistinguishable.
3. Factoring assumption: the simulated Ñ (generated with a known trapdoor) is computationally indistinguishable from a real MPC-generated modulus, since both are products of two large primes.

Therefore, no environment Z can distinguish the real execution from the ideal, and Π UC-realizes F_Ephemeral. ∎

**vs. Stateless AI:** Stateless AI avoids retention but does not cryptographically enforce time. CHRONOS enforces time via VDF sequentiality.

## 6. Discussion and Limitations

### 6.1 Limitations (Explicitly Stated)

1. **Physical attacks are out of scope.** Cold-boot, DMA, and cache-side-channel attacks are not prevented. Hardware countermeasures (TPM, secure enclave) are required for physical security.

2. **Exclusivity Assumption.** The SNARK erasure guarantee assumes no copies of sk exist outside the committed region M. Violations (compiler register spills, OS swap before mlock) break this assumption. Production deployments must enforce mlock/VirtualLock and disable core dumps.

3. **FHE latency.** Real-time TFHE inference for models > 1M parameters requires GPU or FPGA acceleration. CPU-only operation is limited to small models.

4. **drand dependency.** Network connectivity is required for the deadline confirmation. Offline operation requires a trusted local clock (additional hardware assumption).

5. **MPC ceremony cost.** Multi-party modulus generation requires coordination among ≥ 2 independent parties and ~45 seconds of computation per party.

6. **Prototype vs. production gap.** The current implementation uses Paillier HE and hash-chain PoSW as substitutes. The security guarantees of the paper apply to the production configuration (TFHE + Wesolowski VDF + Groth16), not the prototype alone.

### 6.2 Future Work

1. **Post-quantum VDF:** Replace RSA-based Wesolowski with class group VDFs [35] or isogeny VDFs [36].
2. **Formal verification:** Prove the agent stack in seL4 or Coq.
3. **Hardware integration:** Combine CHRONOS with TPM for physical erasure binding.
4. **Dynamic missions:** Extend mission duration via re-encryption under a new VDF without key exposure.
5. **Production backends:** Integrate arkworks-rs/groth16 (Rust) for SNARK proving; integrate gmpy2 + custom squaring loop for Wesolowski VDF.

## 7. Conclusion

We presented CHRONOS, a compositional architecture for ephemeral FHE agents with VDF time-locking and attestable software erasure. Our contributions are: (1) a systematic integration of FHE, VDF, and SNARKs into a single agent lifecycle with formally proven security; (2) the first UC composition theorem for ephemeral FHE agents, with explicit PPT reductions in all hybrid proofs; and (3) a prototype implementation demonstrating feasibility with honest documentation of prototype-vs-production gaps. CHRONOS explicitly does not provide physical erasure guarantees and requires explicit system assumptions for software zeroization. It provides a formally analyzed blueprint for disposable AI agents in high-privacy environments.

## Appendix A: Full UC Proof

### A.1 Simulator Construction

We construct simulator S for the UC proof of Theorem 4.6.

**S.Setup:** Upon receiving (INIT, T_mission, model) from F_Ephemeral:
1. Generate Ñ with known factorization p̃, q̃ (simulation trapdoor; sampled as product of two random 1024-bit primes).
2. Compute φ(Ñ) = (p̃−1)(q̃−1) to enable fast VDF evaluation: ỹ = g^{2^T mod φ(Ñ)} mod Ñ.
3. Generate (pk̃, sk̃) ← TFHE.KeyGen(1^λ).
4. Compute K̃_enc = HKDF(ỹ ‖ salt); ct̃_sk = AES-GCM(K̃_enc, sk̃).
5. Send (Ñ, pk̃, ct̃_sk, params) to Z. Note: Ñ is computationally indistinguishable from a real MPC modulus by the factoring assumption.

**S.Infer:** Forward ct_input to F_Ephemeral; receive and return ct_output to Z.

**S.Deadline:** Receive (ERASED, σ) from F_Ephemeral. Generate π̃_erase using Groth16 zero-knowledge simulator S_Groth16 (public inputs only). Send to Z.

### A.2 Indistinguishability Hybrids

| Hybrid | Change | Indistinguishable by |
|--------|--------|---------------------|
| H₀ | Real protocol | — |
| H₁ | Replace TFHE ciphertexts with random | TFHE CPA security |
| H₂ | Replace SNARK proofs with simulated | Groth16 zero-knowledge |
| H₃ | Replace N with trapdoor-generated modulus | Factoring assumption |
| H₄ | Replace VDF with F_VDF evaluation | VDF zero-knowledge |
| H₅ | Ideal execution under F_Ephemeral | Simulator S |

Each consecutive pair is computationally indistinguishable. Therefore the real execution is indistinguishable from the ideal. ∎
